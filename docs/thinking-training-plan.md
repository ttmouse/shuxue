# 数学口算大闯关 — 思维训练进化方案

> 基于 2026-06-01 讨论结论整理

## 背景与核心理念

### 问题

原项目是一个传统口算练习工具，核心假设是"计算速度 = 数学能力"。在 AI 时代，这个假设需要重新审视——计算可以交给 AI，但**数学思维、数感、分解问题的能力**才是孩子需要训练的东西。

### 目标用户

- 四年级（9~10 岁）孩子
- 有一定的口算基础，正在学习运算定律和应用题
- 对"做题"兴趣不大，但对"游戏/挑战/解谜"有兴趣

### 设计原则

1. **思维路径 > 计算速度** —— 不求算得快，求"怎么想到的"
2. **保护兴趣** —— 做错不惩罚，做对给具体的反馈（不是"你真棒"，而是"你用了分配律，很聪明"）
3. **短时高频** —— 每天 5~8 分钟，到点就停，比 20 题更重要
4. **AI 时代不贬值** —— 训练的是 AI 做不了的事：直觉、模式识别、验证

## 筛选清单：适合做的思维训练

### ★★★★★ 优先实现

| 类型 | 核心思维 | 说明 |
|---|---|---|
| **24 点** | 反向组合 + 试错 | 4 个数，加减乘除凑 24。多种解法，鼓励探索 |
| **运算定律巧算** | 分解 + 替换 | `99×a=100a-a`、`25×a=100a÷4`、`101×a=100a+a` |
| **抽卡凑数** | 目标分解 | 给目标数和几个数字，用运算凑出来。24 点的泛化版本 |

### ★★★★ 后续扩展

| 类型 | 核心思维 | 说明 |
|---|---|---|
| **判断对错** | 估算 + 批判性验证 | AI 给出答案，孩子判断对不对 |
| **找规律填数** | 模式识别 | `2,4,8,?`、`1,4,9,16,?` |
| **数字推理** | 代数直觉 | "两个数相加 15，相乘 56，是哪两个数？" |
| **找错误** | 过程追踪 | "同学作业"某一步错了，让孩子找出来 |

### ★★★ 需更多改造

| 类型 | 核心思维 | 说明 |
|---|---|---|
| **估算题** | 数量级 | 不精确算，判断合理范围 |
| **Fermi 估算** | 量纲推理 | "一个教室能站多少人？"（超出口算，思维价值高） |

## 实现计划（按优先级）

### Phase 1：24 点挑战

- 新模式，独立页面
- 生成 4 个数字，目标凑 24
- 接受算式输入（如 `(3+5)×(4-1)`）
- 支持查看所有解法（给探索感）
- 积分系统：发现多种解法加分

### Phase 2：运算定律巧算专题

- 将现有巧算生成器提升为核心模式
- 每题完成后强制展示拆解过程（"为什么可以这么拆？"）
- 难度阶梯：提示 → 半填空 → 独立完成

### Phase 3：抽卡凑数

- 24 点的泛化：目标不是固定 24，难度可调
- 可选的运算符范围
- 自由模式任意组合

### Phase 4（远期）：判断对错 / 找规律 / 数字推理

- 各自独立，UI 差异较大，需要单独设计

## 交互体验设计要点

### 反馈系统（和传统模式不同）

| 场景 | 旧模式 | 新模式 |
|---|---|---|
| 做对 | ✅ + "你真棒" | ✅ + "你用分配律拆开了 99×34，很棒"（具体反馈） |
| 做错 | ❌ + 记错题本 | 💡 + "差一点！看看这两个数有什么关系？"（引导提示） |
| 全部完成 | 正确率 + 错题重练 | "今天你用了 3 种不同的方法来凑 24，越来越灵活了" |
| 每天 | 无限做 | 做完 5 题或 5 分钟 → "今天的训练完成了，明天再来"（有终结感） |

### 时长控制

- 不再以"完成 N 题"为单位
- 改为"完成当日挑战"（5 题 / 凑出 1 组 24 点 / 5 分钟）
- 到点告诉孩子"今天够了"，保持期待感

### 游戏化元素

- 不需要登录/排行榜——专注个人成长
- 连续完成天数展示（"今天是你连续练习的第 X 天"）
- 发现新解法数量计数（量变积累成就感）

## 不做的方向（明确排除）

- 不做登录/用户系统（localStorage 够用）
- 不做排行榜/竞赛（避免焦虑）
- 不做大数竖式计算（AI 时代低价值）
- 不做速度计时器（和核心理念冲突）
- 不引入任何外部依赖（保持零依赖、离线可用）
