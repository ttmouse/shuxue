# 数学思维训练 — 框架设计

> 设计目标：一个可扩展的多模式思维训练平台，采用多林国（Duolingo 风格）视觉语言，聚焦"思维路径"而非"计算速度"。

---

## 一、整体架构

### 1.1 页面结构

```
首页（模式选择）
  ├── 🎯 24 点挑战      ← Phase 1
  ├── ⚡ 运算定律巧算    ← Phase 2
  ├── 🃏 抽卡凑数        ← Phase 3
  └── 📚 传统口算        ← 保留原有模式
```

### 1.2 模式注册契约

每个训练模式是一个独立的 JS 文件，通过统一接口注册到框架：

```js
// 模式必须暴露的接口
{
  id: '24game',             // 唯一标识
  name: '24 点挑战',        // 显示名称
  icon: 'dice',             // Lucide 图标名

  // 页面渲染
  render(container) {},     // 渲染模式界面
  destroy() {},             // 离开时清理

  // 可选
  onPageShow() {},          // 切回时恢复
  onPageHide() {},          // 离开时暂停
}
```

框架负责：导航、页面切换、动画过渡、sound、stats 公用服务。

## 二、设计系统（多林国风格）

基于 Mini-Habits 项目的设计系统提炼，调整为数学训练的主题色。

### 2.1 颜色

```
--n50:~f7f6f3    --n100:#f2f0eb   --n150:#e8e5df
--n200:#e4e4e7   --n250:#d4d1ca   --n300:#a8a49d
--n400:#8a8680   --n500:#6d6964   --n600:#55514c
--n700:#3c3c3c   --n800:#27272a   --n900:#1a1a18

--primary:#58CC02        ← Duolingo 绿（数学 = 成长）
--primary-light:#46a302
--primary-dark:#3a8a02

--blue:#1cb0f6           ← 逻辑/推理
--orange:#ff9600         ← 创意/巧算
--purple:#c054f7         ← 挑战/成就
--red:#FF4B4B            ← 错误/注意
--gold:#ff9600           ← 奖励
```

数学主题选绿色为主色（不同于 Mini-Habits 的紫色/橙色混合），因为：
- 绿色在数学语境中代表"正确/成长"，视觉上温和
- 和 Duolingo 的关联度最高，孩子有熟悉感

### 2.2 按钮系统（3D 风格）

```css
.btn-primary {
  background: linear-gradient(135deg, var(--primary), var(--primary-light));
  color: #fff;
  box-shadow: 0 4px 0 var(--primary-dark), 0 6px 12px rgba(88,204,2,0.3);
  /* 按下时：transform: translateY(2px); box-shadow 改为 0 1px 0 */
}
```

### 2.3 模式选择卡片

每个模式用大卡片展示，包含：图标、名称、简短描述、难度标签。

```
┌─────────────────────────────────┐
│  🎯  24 点挑战                   │
│  用 4 个数字加减乘除凑出 24       │
│  [初级] [中级] [高级]            │
└─────────────────────────────────┘
```

卡片特征：大白底、12px 圆角、hover 时绿色边框、进场上滑动画。

### 2.4 动效语言

| 场景 | 动画 |
|---|---|
| 页面进入 | `translateY(12px) → 0` + fade，弹簧曲线 |
| 按钮按下 | `scale(0.97)` 或 3D 下沉 |
| 答对 | 绿色闪光 + 浮动 "+1" 文字 |
| 答错 | 轻柔 shake |
| 完成 | 庆祝动画（五彩纸屑） |
| 卡片出现 | 交错上滑（`nth-child` 延迟） |

- 统一的**顶栏**（返回按钮 + 模式名称 + 状态）
- 统一的**底部安全区**
- 统一的**反馈层**（toast / 弹窗）
